# Repository Audit: craftalism-deployment

Date: 2026-04-10

## Repository Role Summary

`craftalism-deployment` is the runtime composition and environment alignment
layer for the Craftalism platform. It owns Docker Compose orchestration,
environment consistency, cross-service wiring, and operator-facing automation
for local, test, and production flows.

## Requirement Audit

### Owned

- `runtime/environment alignment rules`: partially compliant
  - Compose wiring, localhost production bindings, and issuer propagation are
    mostly coherent.
- `deployment-side enforcement/validation`: partially compliant
  - Production preflight checks only presence of variables, not whether they
    still contain placeholder or invalid values.

### Consumed

- `auth-issuer`: compliant
  - `AUTH_ISSUER_URI` is required and propagated consistently to auth, API, and
    Minecraft runtime wiring.
- `ci-cd`: partially compliant
  - Workflow validation exists, but PR execution is broken by incorrect ref
    selection for cross-repo builds/checkouts.
- `testing`: partially compliant
  - A meaningful smoke path exists and covers auth, protected API write, API
    read-back, and dashboard proxy read.
- `documentation`: compliant
  - Docs broadly match current scripts and compose overlays.
- `security-access-control`: partially compliant
  - Exposure assumptions are explicit, but production validation still accepts
    placeholder secrets and digest values.

### Local-Only Responsibilities

- Compose file consistency: compliant
- Overlay/environment strategy: compliant
- Health checks and startup ordering: compliant
- Helper scripts for local/test/prod operations: partially compliant
- Runtime defaults and env propagation: partially compliant
- Deployment documentation and reproducibility discipline: compliant

### Governance

- No evidence of application-business-logic ownership drift into service repos.
- Cross-service runtime consistency is treated as a first-class concern, but the
  production preflight and PR CI path still leave avoidable operational gaps.

## Confirmed Problems

### 1. High: PR workflow uses the wrong ref for cross-repo builds

The staging workflow uses `${{ github.ref_name }}` as the branch/ref when
building images from external service repositories and when checking out the
economy repo. On `pull_request` events, `github.ref_name` resolves to
`<pr_number>/merge`, not the source branch name, so those refs will not exist
in sibling repositories. That breaks PR validation for both remote Docker build
contexts and the economy checkout path.

Evidence:

- [build-staging-images.yml](/home/henriquemichelini/IdeaProjects/craftalism-deployment/.github/workflows/build-staging-images.yml#L163)
- [build-staging-images.yml](/home/henriquemichelini/IdeaProjects/craftalism-deployment/.github/workflows/build-staging-images.yml#L181)

Impact:

- PR workflows can fail before meaningful deployment validation runs.
- CI confidence is weaker than the repo documentation implies.

### 2. Medium: production preflight accepts placeholder secrets and digest values

`./prod` checks only that required variables are non-empty. If `.env` is copied
from `env.example`, placeholder values such as `change_me_in_production`,
`YOUR_BASE64_KEY_HERE`, and `sha256:replace_with_release_digest` pass the
preflight even though they are not deployable or safe.

Evidence:

- [prod](/home/henriquemichelini/IdeaProjects/craftalism-deployment/prod#L34)
- [prod](/home/henriquemichelini/IdeaProjects/craftalism-deployment/prod#L73)
- [env.example](/home/henriquemichelini/IdeaProjects/craftalism-deployment/env.example#L3)

Impact:

- Fail-fast guarantees are weaker than documented.
- Operators can get deeper into a deployment with invalid configuration than
  they should.

## Recommended Repo-Local Implementation Scope

- Make workflow ref selection event-aware:
  - use `github.head_ref` for `pull_request`
  - use `github.ref_name` for `push` and `workflow_dispatch`
- Strengthen `./prod` validation to reject:
  - placeholder secret values
  - placeholder digest values
  - obviously invalid issuer URLs
  - placeholder RSA key material

## Optional Improvements

- Add stricter `.env` preflight checks for invalid combinations and hostname
  consistency.
- Record deployment evidence after successful test or production runs.
- Add rollback guidance for digest-based production deployments.
